
# 1. Complete Artifact Persistence Pipeline

Your problem occurs because artifacts are treated as **temporary streaming events**, but in production systems they must be **persisted entities**.

Modern AI platforms (ChatGPT, Claude, Cursor, Perplexity) use a **three-layer pipeline**.

```
Agent Runtime
      ↓
Artifact Storage Layer
      ↓
Chat Message Metadata
      ↓
Frontend Renderer
```

---

# 2. Correct Artifact Lifecycle

The correct lifecycle is:

```
Agent generates artifact
↓
Artifact file saved to disk
↓
Artifact record stored in database
↓
Artifact linked to ChatMessage
↓
Artifact streamed to UI
↓
Artifact available on history reload
```

Streaming events are **only for live updates**, not storage.

---

# 3. Backend Storage Architecture

You must persist artifacts in **three places**.

### File Storage

Artifacts stored on disk.

Example path:

```
data/artifacts/{artifact_id}.{ext}
```

Examples:

```
data/artifacts/abc123.png
data/artifacts/def456.csv
data/artifacts/ghi789.json
```

---

### Database Table

Artifacts must have a dedicated table.

```
Artifact

id
user_id
session_id
message_id
filename
mime_type
file_path
created_at
```

Purpose:

* track ownership
* track location
* link artifact to message

---

### Chat Message Metadata

Artifacts must be referenced in the message metadata.

Example message record:

```
ChatMessage

id
session_id
role
content
metadata
created_at
```

Metadata example:

```
{
  "artifacts": [
    {
      "id": "abc123",
      "type": "image",
      "url": "/api/artifacts/abc123"
    }
  ]
}
```

---

# 4. Artifact Generation Pipeline

Correct backend pipeline:

```
Agent tool execution
↓
Artifact generated
↓
Save artifact file to disk
↓
Insert artifact record in DB
↓
Attach artifact to ChatMessage metadata
↓
Emit SSE artifact event
```

Example SSE event:

```
event: artifact
data: {
  "id":"abc123",
  "type":"image",
  "url":"/api/artifacts/abc123"
}
```

This allows:

* live display
* persistent storage

---

# 5. Artifact API

Frontend must load artifacts through API.

Endpoint:

```
GET /api/artifacts/{artifact_id}
```

Responsibilities:

```
validate user ownership
load file from disk
return correct mime type
stream file
```

Example:

```
Content-Type: image/png
```

---

# 6. Chat History Loading

When frontend loads chat history:

```
GET /chat/{session_id}/history
```

Response must include artifacts.

Example:

```
{
  "messages": [
    {
      "role": "assistant",
      "content": "Here is the chart",
      "artifacts": [
        {
          "type": "image",
          "url": "/api/artifacts/abc123"
        }
      ]
    }
  ]
}
```

# 8. Refresh Behavior

Correct refresh behavior:

```
User refresh page
↓
Frontend loads chat history
↓
Artifacts included in message metadata
↓
Frontend renders artifacts again
```

Artifacts never disappear.

# 9. Multi-language `/code` Pipeline

Your `/code` system must follow this architecture.

```
User runs /code
↓
AI generates code
↓
Frontend shows code editor
↓
User clicks run
↓
Backend executes code runner
↓
Console output returned
```

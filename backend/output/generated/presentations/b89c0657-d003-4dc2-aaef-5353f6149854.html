Web Services & Cloud Computing Foundations

Web Services, SOA, and Cloud Computing are interrelated but independent paradigms. Web Services act as the communication mechanism enabling SOA and Cloud Computing.

  • Cloud Computing is a broad term covering any Web service software, hardware, and applications.
  • SOA focuses primarily on software services and service-oriented architecture.
  • Web Services encapsulate Cloud services for network connections using standard protocols.

Remote Procedure Call (RPC) Architecture

Client-Server Communication via Stubs and Marshaling

Image: Split-screen cyberpunk illustration: left side shows client machine with glowing neon cyan RPC client process, right side shows server machine with neon pink RPC server process. A glowing data packet travels across a network link. Stubs are depicted as translucent holographic interfaces with XML-like data streams. Background: dark grid with pulsing data nodes.

RPC enables a process on one machine to invoke a procedure on another via stub functions and network communication.

  • Client calls the client stub, which marshals parameters and sends a message over the network.
  • Server stub unmarshals parameters and invokes the server function.
  • Server returns result; server stub marshals and sends response back.
  • Client stub unmarshals return value and returns to client code.

Web Services Architecture Stack

XML, WSDL, and UDDI: The Foundational Triad

Image: Layered cyberpunk architecture diagram: bottom layer labeled 'XML' (glowing neon cyan text), middle layer labeled 'WSDL' (neon pink), top layer labeled 'UDDI' (neon cyan). Arrows show data flow upward. Background: dark grid with floating XML tags, service registry icons, and WSDL schema fragments.

Web services rely on XML-based technologies for messaging, description, and discovery.

  • XML: Platform-independent, text-based data format enabling cross-platform communication.
  • WSDL: XML-based language defining service interface, operations, parameters, and location.
  • UDDI: Universal Description, Discovery, and Integration registry for publishing and discovering services.

Cloud Computing: Core Characteristics

Essential Traits Enabling Scalable, On-Demand Delivery

Image: Cyberpunk cloud icon with glowing neon pink outline, floating above a dark grid. Six holographic labels orbit the cloud: 'On-demand Self-Service', 'Broad Network Access', 'Resource Pooling', 'Rapid Elasticity', 'Measured Service', 'Flexible Billing'. Background: abstract server racks with pulsing data streams.

Cloud computing delivers on-demand resources over the network with key characteristics enabling scalability and cost-efficiency.

  • On-demand self-service: Users automatically acquire/release resources without provider intervention.
  • Broad network access: Resources available over standard networks (e.g., HTTP) via thin/thick clients.
  • Resource pooling: Physical/virtual resources pooled and dynamically assigned (multi-tenant model).
  • Rapid elasticity: Resources scale instantly and automatically to match demand (appear unlimited).
  • Measured service: Resource consumption monitored, controlled, and reported for optimization.
  • Flexible billing: Pay-as-you-go or subscription-based pricing tied to actual usage.

Virtualization: Hypervisor Types

Type 1 (Bare-Metal) vs. Type 2 (Hosted): Performance vs. Accessibility

Image: Split-screen cyberpunk comparison: left side labeled 'Type 1 Hypervisor' with neon pink border, bare-metal server icon, and hypervisor directly on hardware; right side labeled 'Type 2 Hypervisor' with neon cyan border, hypervisor on OS layer, and host OS icon. Background: dark grid with virtualization layer overlays.

Hypervisors enable multiple VMs on a single host. Type 1 offers higher performance; Type 2 prioritizes ease of use.

  • Type 1 (Bare-Metal): Hypervisor runs directly on hardware (e.g., VMware ESXi, Xen). Higher performance, less overhead.
  • Type 2 (Hosted): Hypervisor runs on host OS (e.g., VMware Workstation, Hyper-V). Easier to install, more points of failure.
  • Performance Trade-off: Type 1 supports more VMs per host; Type 2 has OS layer overhead reducing efficiency.
  • Use Case: Type 1 for production clouds; Type 2 for development/testing environments.

Distributed systems must achieve transparency, scalability, and resilience despite heterogeneity and failures.

ConcurrencyManages shared resources via fair scheduling, dependency preservation, deadlock avoidance

Fault ToleranceContinues operation despite component faults (failure = non-compliance; fault = cause)

Web Services vs. SOA vs. Cloud Computing

Web Services, SOA, and Cloud Computing are independent approaches, though Web Services often serve as the connection mechanism for Cloud Computing.

  • Cloud Computing: Broad term for any Web service software, hardware, and applications.
  • SOA: Focuses mainly on software services and service-oriented architecture.
  • Web Services: Encapsulate Cloud services; used for connections in Cloud Computing.
  • Web Services can be used outside Cloud Computing, as part of a service-oriented architecture.

RPC: Remote Procedure Call Architecture

RPC enables distributed computation by allowing a client process to invoke a procedure on a remote server through stubs and network communication.

Image: Split-screen diagram: left side shows client process calling stub, right side shows server process with skeleton. Arrows labeled 'marshal', 'network', 'unmarshal' in neon pink and cyan. Background dark with subtle wireframe grid.
  • Client calls client stub, which marshals parameters and sends message over network.
  • Server stub unmarshals parameters and calls server function.
  • Server returns result; stub marshals and sends response back to client stub.
  • Client stub unmarshals return value and returns to client code.

Web Services Architecture

Web services solve interoperability, firewall traversal, and complexity issues of earlier distributed systems like RMI and CORBA.

  • Interoperability: Web services use HTTP and XML, avoiding vendor-specific on-wire formats.
  • Firewall Traversal: Use standard HTTP port 80, enabling easier collaboration across firewalls.
  • Reduced Complexity: Developer-friendly compared to RMI/CORBA learning curves.
  • Platform independence: Language-neutral protocols enable cross-platform communication.

Web Services Stack & XML

Web services rely on XML-based technologies for messaging, description, and registry functions.

  • XML Messaging: SOAP uses XML for remote calls across heterogeneous systems.
  • WSDL: XML-based language defining service interface, parameters, protocols, and location.
  • UDDI: XML-based registry for publishing and discovering web services.
  • XML is pure text, enabling cross-platform data exchange without binary compatibility issues.

Virtualization & Hypervisors

Virtualization decouples OS from hardware, enabling migration, isolation, and efficient resource utilization.

  • Virtual Machine: Software implementation of a machine executing programs like a physical machine.
  • Hypervisor: Thin software layer enabling multiple OS to share hardware (Type 1: bare-metal; Type 2: hosted).
  • Full Virtualization: Guest OS unaware; uses binary translation or direct execution.
  • Para-Virtualization: Guest OS modified for optimization; stub drivers improve performance.

Distributed Systems & Middleware

Distributed systems appear as a single coherent system, with middleware masking heterogeneity and enabling transparent communication.

  • Transparency: Hides distribution aspects (access, location, migration, replication, failure).
  • Openness: Enables interoperability across heterogeneous environments.
  • Scalability: System remains effective with increased users and resources.
  • Middleware: Software layer above OS, below apps, masking heterogeneity (e.g., RPC, CORBA).

Cloud Deployment Models

  • Public Cloud: Accessible to general public; less secure but cost-effective (e.g., AWS, Azure).
  • Private Cloud: Exclusive to single organization; enhanced security and control.
  • Community Cloud: Shared by organizations with common concerns (e.g., security, compliance).
  • Hybrid Cloud: Mix of public and private; critical tasks use private, non-critical use public.

Hybrid clouds enable 'cloud bursting'—releasing public cloud resources during sudden demand spikes.

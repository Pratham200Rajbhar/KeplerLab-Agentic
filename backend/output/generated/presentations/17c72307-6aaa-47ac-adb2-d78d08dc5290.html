Infrastructure as a Service (IaaS)

Core Concepts and Architectural Foundations

‘All cloud computing services provide access to computing resources in a virtualized environment, the Cloud, across a public connection, usually the internet.’
IaaS delivers virtualized hardware infrastructure—virtual machines, storage, networking—over the internet or private networks.
  • Virtualized hardware resources (CPU, memory, storage, networking)

Key Characteristics of IaaS

Self-Service, Metering, and Dynamic Scaling

‘You can’t discuss the key characteristics of IaaS without understanding the imperative of self-service.’
Self-service enables users to provision resources on-demand—like ATMs for IT infrastructure.
  • Self-service provisioning: users request and deploy resources without manual intervention

IaaS Resource Types and Virtualization

What Users Actually Access

Resource TypeDescription
Virtual MachinesFully configurable compute instances
Virtual StorageScalable disk storage (e.g., VM disk images)
Virtual NetworksVLANs, load balancers, IP addresses
Software BundlesPre-configured OS and middleware stacks

Resources are virtualized and pooled from underlying physical infrastructure, enabling abstraction and elasticity.

Private Cloud and Hybrid Cloud Models

Controlled Environments for Sensitive Workloads

Private clouds offer public-cloud functionality on private infrastructure—within firewall, with full control over data and governance.
  • Private cloud: dedicated internal environment, supports self-service provisioning and automated virtualization

Live Migration and VM Lifecycle Management

Enabling Resilience and Efficiency

‘Live migration is the movement of a virtual machine from one physical host to another while powered on—without noticeable effect to the end user.’
Live migration enables proactive maintenance and load balancing—critical for high-availability systems.
  1. VM state captured as portable data files (CPU, memory, disks, network)

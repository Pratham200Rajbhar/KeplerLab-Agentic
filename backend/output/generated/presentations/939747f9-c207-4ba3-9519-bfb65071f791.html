Distributed Systems Fundamentals

Core Concepts and Architectural Layers

  • A collection of interconnected, stand-alone computers working cooperatively as a single integrated computing resource
  • Distributed systems mask heterogeneity via middleware
  • Services are specified through well-defined interfaces, often in Interface Definition Languages
Image: A diagram showing multiple interconnected computers (nodes) with arrows indicating communication paths, overlaid with a translucent software layer labeled 'Middleware' that abstracts OS and hardware differences

Transparency in Distributed Systems

Transparency TypeDescription
AccessHide differences in data representation and how a resource is accessed
LocationHide where a resource is located
MigrationHide that a resource may move to another location
ReplicationHide that a resource is replicated
FailureHide the failure and recovery of a resource

Transparencies collectively enable users and applications to interact with distributed resources without needing to understand underlying complexity.

Middleware: The Distributed Systems Glue

  • A software layer between applications and the OS, masking heterogeneity of underlying components
  • Provides a common programming abstraction across a distributed system
  • Manages complexity and heterogeneity inherent in distributed systems
  • Enables application portability and interoperability across heterogeneous environments

XML vs HTML: Purpose and Structure

FeatureXMLHTML
Tag ExtensibilityUser-defined, extensible tagsFixed, pre-determined tags
OrientationContent-orientedPresentation-oriented
Primary UseTransport and store dataDisplay data
ValidationSupports schema validation (XSD)No built-in validation

XML is designed for data transport and storage with strict syntax and validation; HTML is designed for human-readable presentation.

XML Document Validity

  • Well-formed: Syntax is correct — all tags opened and closed properly
  • Valid: All elements conform to definitions in the schema (XSD)
  • An XML instance document must satisfy both well-formedness and validity constraints to be accepted by compliant parsers

Cloud computing delivers computing, storage, and software as utilities accessible via the internet.

Cloud Computing Principles

  • Virtualized, dynamically provisioned computers presented as unified computing resources
  • Based on service-level agreements (SLAs)
  • Services are specified through interfaces (e.g., IDLs)
  • Emphasizes loose coupling, standard-based interoperability, scalability, and reusability
Image: A cloud icon with internal virtual machines, containers, and network components, connected to users via internet, with SLA document and interface icons floating nearby

Service-Oriented Computing (SOC)

  • Core reference model for cloud computing systems
  • Services are the fundamental building blocks of applications and systems
  • Supports rapid, low-cost, flexible, interoperable, and evolvable development
  • Enables application-to-application communication and programmatically accessible services

SOC underpins modern distributed and cloud architectures by promoting modularity and reuse.

UDDI: Web Services Registry

  • Universal Description, Discovery, and Integration (UDDI) serves as a business and service registry
  • Enables businesses to populate service descriptions for easier integration over the Web
  • Facilitates discovery of services by consumers and partners
  • Essential for widespread adoption and interoperability of Web services

Key Distributed Systems Characteristics

  • Loose Coupling: Components interact via well-defined interfaces
  • Standard-Based: Ensures interoperability across platforms
  • Flexibility & Reusability: Services can be composed and reused
  • Scalability: Systems can grow horizontally without redesign
  • Reduced Complexity: Middleware abstracts heterogeneity
  • Programmatically Accessible: Services expose APIs for integration

Infrastructure as a Service (IaaS)

Core Concepts, Architecture, and Deployment Models

What is IaaS?

IaaS provides virtualized computing resources over the internet. Physically, hardware resources are pulled from servers and networks distributed across multiple data centers—maintained by the cloud provider. Clients access virtualized components to build their own IT platforms.

  • Dynamically provisioned via public web services (REST/SOAP)
  • Utility billing: users pay only for used capacity at a given time
  • Provider manages hardware; user manages OS, middleware, apps

Key Characteristics of IaaS

  • On-demand self-service provisioning
  • Broad network access (e.g., via REST APIs)
  • Resource pooling and multi-tenancy
  • Rapid elasticity (scaling up/down)
  • Measured service (pay-per-use)

Private & Hybrid Cloud IaaS

  • Private IaaS: Eucalyptus, OpenNebula
  • Hybrid cloud: Combines private and public resources
  • Cloud bursting: Uses public cloud to handle sudden demand spikes

Virtual Machine Provisioning Process

  • Select a physical server with sufficient capacity and an OS template
  • Load OS, drivers, middleware, and required applications
  • Customize configuration (IP, gateway, network, storage)
  • Deploy and start the virtual server

VM Migration Techniques

  • Hot / Live migration: VM remains running during move
  • Cold / Regular migration: VM is powered off during move
  • Live storage migration: Move storage while VM runs

All VM components (CPU, memory, storage, networking) are virtualized, enabling full state capture as portable data files.

Virtual Data Centers (VDC)

  • A virtualized network of interconnected virtual servers
  • Used for enhanced cloud hosting, enterprise IT infrastructure
  • Deployable in private, public, or hybrid cloud environments

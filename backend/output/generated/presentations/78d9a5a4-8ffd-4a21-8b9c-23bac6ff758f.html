Distributed Systems & Cloud Computing Foundations

Core Concepts, Architectures, and Service Models

A distributed system is a collection of interconnected stand-alone computers working cooperatively as a single integrated computing resource [Buyya].
Image: A network diagram showing multiple interconnected computers (nodes) with arrows indicating bidirectional communication, forming a unified cloud-like structure labeled 'Single Integrated Resource'. Background gradient from light blue to white.

Key Design Principles of Distributed Systems

  • Loose Coupling: Minimizes dependencies between components
  • Standard-Based: Ensures interoperability via open standards
  • Flexibility: Adapts to changing requirements and environments
  • Reusability: Components can be reused across systems
  • Scalability: Supports growth in users, data, or resources
  • Reduced Complexity: Abstracts heterogeneity from developers
Systems must conform to well-defined interfaces, support application portability, and enable easy interoperability.

Transparency in Distributed Systems

Transparency TypeDescription
AccessHide differences in data representation and resource access
LocationHide where a resource is located
MigrationHide that a resource may move to another location
ReplicationHide that a resource is replicated
ConcurrencyHide that a resource may be shared by several competitive users
FailureHide the failure and recovery of a resource
PersistenceHide whether a resource is in memory or on disk
Transparencies collectively mask complexity, enabling developers to focus on logic rather than infrastructure behavior.

Middleware: The Distributed Systems Glue

‘a software layer between applications and the OS whose purpose is to mask heterogeneity of the underlying components and to provide a convenient programming model to application programmers.’
  • Manages complexity and heterogeneity in distributed systems
  • Resides above OS but below application programs
  • Provides common programming abstraction across distributed environments
  • Enables seamless communication and resource sharing

Service-Oriented Computing (SOC)

‘Service orientation is the core reference model for cloud computing systems.’
  • Services are main building blocks of applications/systems
  • Supports rapid, low-cost development
  • Enables flexible, interoperable, and evolvable systems
  • Forms foundation of cloud middleware and service interfaces
SOC underpins the design of modern distributed and cloud systems.

Cloud Computing Types & Deployment Models

  • Private Cloud: Owned by single organization, shared internal missions
  • Community Cloud: Shared by organizations with common concerns (e.g., security)
  • Public Cloud: Available to general public, owned by third-party provider
  • Hybrid Cloud: Combines two or more models (e.g., private + public)
Image: Venn diagram with four overlapping circles labeled 'Private', 'Community', 'Public', and 'Hybrid'. Hybrid circle overlaps all others. Use muted blue for private, amber for community, green for public, and purple for hybrid. Background white.

Web Services Registry: UDDI

‘Universal Description, Discovery and Integration (UDDI) serves as a business and service registry essential for the widespread use of Web services.'
  • Central registry for publishing and discovering web services
  • Businesses populate descriptions of their services
  • Enables service integration over the Web
  • Supports dynamic discovery and binding of services
UDDI is foundational for service-oriented architectures in distributed environments.

Cloud middleware abstracts infrastructure complexity and enables scalable application deployment.

Cloud Computing Needs & Enablers

  • Virtualization: Enables resource abstraction and pooling
  • Cloud Architecture: Supports elasticity and multi-tenancy
  • Service Models: IaaS, PaaS, SaaS define delivery boundaries
  • Management & Monitoring: Ensures SLA compliance and performance
  • Security: Addresses data privacy, access control, compliance
Virtualization is the foundational technology enabling cloud computing scalability and efficiency.

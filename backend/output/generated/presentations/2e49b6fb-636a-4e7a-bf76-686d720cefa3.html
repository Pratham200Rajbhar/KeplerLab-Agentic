Remote Procedure Call (RPC) Architecture

RPC enables distributed computation by abstracting network communication through stubs—client-side stub marshals parameters, server-side stub unmarshals and invokes the remote function.

  • Client stub marshals parameters and sends message over network to server stub
  • Server stub unmarshals parameters and invokes actual server function
  • Return value is marshaled by server stub, transmitted back, and unmarshaled by client stub

RMI Architecture & Development Lifecycle

RMI enables Java objects to invoke remote methods transparently. The registry binds server names, clients lookup references, and stubs/skeletons handle marshaling/unmarshaling.

  • Server binds remote object to registry name; client looks up and obtains stub reference
  • Stub serializes parameters and sends to skeleton; skeleton unmarshals and invokes remote method
  • Return value is serialized back through stub to client
StepAction
1Define remote interface
2Implement remote object

Web Services Architecture & XML Foundation

Web Services rely on XML for messaging, description (WSDL), and discovery (UDDI). XML enables platform-neutral data exchange and structured communication.

  • XML is pure text—no binary data—allowing any application to parse and share data regardless of platform
  • WSDL defines service interface, operations, bindings, and endpoint address
  • UDDI serves as a business/service registry for publishing and discovering Web Services

IaaS: Infrastructure as a Service

IaaS provides virtualized computing resources over the internet—virtual machines, storage, networks—enabling scalable infrastructure without hardware procurement.

  • Resources are dynamically scaled, metered, and billed on utility pricing (e.g., $0.02–$2.60/hour per instance)
  • Self-service provisioning allows users to deploy VMs without IT intervention
  • Supports enterprise infrastructure, cloud hosting, and virtual data centers

Virtualization: Hypervisors & VM Migration

Virtualization decouples OS from hardware via hypervisors. Type 1 (bare-metal) offers better performance; Type 2 (hosted) prioritizes ease of use.

  • Full Virtualization: unmodified guest OS (e.g., VMware ESXi, KVM); uses binary translation or direct execution
  • Para-Virtualization: modified guest OS (e.g., Xen); better performance but requires kernel changes
  • Live Migration: moves VMs between hosts with minimal downtime (e.g., VMware VMotion, XenMotion)

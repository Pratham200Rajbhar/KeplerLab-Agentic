Distributed & Cloud Computing Fundamentals

Core Concepts, Architectures, and Service Models

  • Distributed systems: interconnected stand-alone computers working cooperatively as a single resource [Buyya]

Cloud computing delivers computing, storage, and software as utilities over the internet, based on service-level agreements [Buyya].

Key Dimensions of Distributed Systems

  • Operating systems: Multi-node OS support, resource management across nodes, fault tolerance mechanisms
  • Hardware architectures: Heterogeneous compute (CPU/GPU/FPGA), network topologies (e.g., fat-tree, torus), latency-aware placement
  • Communication architectures: Protocols (e.g., gRPC, REST, MPI), consistency models (e.g., eventual, strong), QoS guarantees
  • Programming languages: Language runtime support for concurrency, serialization frameworks, distributed libraries (e.g., Akka, Ray)
  • Software interfaces: APIs (e.g., REST, GraphQL), service discovery, load balancing strategies
  • Security measures: Authentication (e.g., OAuth, mTLS), encryption (in transit/at rest), zero-trust architectures
  • Information representation: Data formats (e.g., Protobuf, Avro), schema evolution, data partitioning and sharding strategies
Each dimension introduces critical trade-offs—such as consistency vs. availability, performance vs. complexity—that must be carefully balanced during design and deployment.

Service-Oriented Computing (SOC)

  • Service orientation is the core reference model for cloud computing systems
  • Supports rapid, low-cost, flexible, interoperable, and evolvable systems
  • Services are the main building blocks of application and system development
  • Services are self-contained and loosely coupled components
  • Enables composition and reuse of existing services to build new applications
  • Supports platform independence through standardized interfaces (e.g., SOAP, REST)
  • Facilitates scalability and fault tolerance via distributed service deployment
  • Promotes business agility by enabling quick adaptation to changing requirements
Service orientation is the core reference model for cloud computing systems.

Web Services Architecture (WSDL Elements)

  • <types>: Provides data type definitions
  • <service>: Aggregates a set of related ports
  • <serviceType>: Mechanism to aggregate portTypes
  • <message>: Represents the abstract definition of data being transmitted
  • <portType>: Defines a set of abstract operations
  • <binding>: Specifies concrete protocol and data format specifications for operations
  • <port>: Specifies an address for a binding
WSDL defines the interface of a Web service, enabling interaction between requester and provider [Excerpt 15].

XML Document Validity

  1. Well-formed: Syntax is correct (all tags opened and closed)
  2. Valid: All elements match definitions in the schema (xsd)
  3. XML Instance Document (xml) == XML Instances of the Schema (xsd)
Well-formedness is syntactic; validity is semantic and schema-conformant.

Middleware: Definition & Role

A software layer between applications and the OS whose purpose is to mask heterogeneity of the underlying components and to provide a convenient programming model to application programmers.
  • Manages complexity and heterogeneity in distributed systems
  • Provides common programming abstraction across distributed systems
  • Resides above OS but below application programs
Middleware enables seamless integration across heterogeneous platforms.

Cloud Computing Characteristics

  • Loose Coupling: Systems interact via well-defined interfaces without tight dependencies
  • Standard-Based (interoperability): Adheres to open standards for cross-platform compatibility
  • Flexibility: Adapts to varying workloads and usage patterns dynamically
  • Reusability: Components and services can be reused across applications and domains
  • Scalability: Resources scale elastically to meet demand, both up and down
  • Reduced Complexity: Abstracts infrastructure details from end users and developers
  • Programmatically Accessible: APIs enable programmatic control and automation
  • Application-to-Application Communication: Designed for machine-to-machine interaction, not just human interfaces
These characteristics enable robust, scalable, and maintainable cloud-based systems.

Web Services Registry: UDDI

UDDI serves as a 'business and service' registry essential for the widespread use of Web services [Excerpt 12].
  • Universal Description, Discovery and Integration (UDDI) is the registry standard
  • Enables interoperability across heterogeneous systems
  • Businesses populate the registry with service descriptions
  • Supports service discovery and integration over the Web

Cloud Middleware Layers

  1. Software As a Service (SaaS): Fully deployed applications accessible via web browsers or APIs
  2. Infrastructure As a Service (IaaS): Virtualized computing resources over the internet
  3. Platform As a Service (PaaS): Development and deployment environment for applications
  4. MapReduce Programming: Distributed data processing paradigm for large-scale analytics
  5. Management and Monitoring: Tools for provisioning, orchestration, performance, and health monitoring
  6. Security: Authentication, authorization, encryption, and compliance enforcement
  7. Cloud Middleware: Unified abstraction layer enabling service composition, lifecycle management, and cross-layer integration
Middleware enables abstraction, orchestration, and management across all cloud service layers.

Cloud Computing Needs & Enablers

  • Virtualization: Core enabler of resource pooling and abstraction
  • Cloud Architecture: Scalable, modular design patterns
  • Types of Virtualization: OS virtualization (hypervisors), hardware virtualization (ISA abstraction), network virtualization (SDN), storage virtualization (unified storage pools)
  • Opportunities and Challenges: Economic trade-offs (CAPEX vs OPEX), technical challenges (latency, consistency), security and privacy concerns
  • Future: Edge integration (decentralized compute), AI-native clouds (auto-scaling, intelligent scheduling), sustainability focus (energy-efficient data centers)
Virtualization is the foundational technology enabling cloud elasticity and scalability.

Interoperability Requirements

  • Systems conform to well-defined interfaces: APIs, protocols, and data formats must be standardized and documented
  • Support portability of applications across environments: Applications should run consistently across different cloud providers or on-premises systems
  • Enable easy interoperability with other open systems: Systems must integrate with external services, databases, and third-party tools without vendor lock-in
Be able to interact with services from other open systems, irrespective of the underlying environment.

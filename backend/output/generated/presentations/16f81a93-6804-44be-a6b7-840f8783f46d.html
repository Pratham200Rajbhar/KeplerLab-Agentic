XML vs HTML, SOAP, and Distributed System Interoperability

XML vs HTML: Core Differences

  • XML: Content-oriented, extensible tags, designed to transport/store data, supports validation
  • HTML: Presentation-oriented, fixed tags, designed to display data, no validation

XML Document Structure

An XML instance includes an XML declaration (e.g., <?xml version="1.0" encoding="UTF-8"?>), optional comments, namespace declarations (xmlns), and a hierarchical element structure. Example: <books xmlns="somename" xsi:schemaLocation="...">...</books>.

SOAP Message Structure

  • Envelope: Root element defining the start/end of the message (xmlns:SOAP-ENV="http://schemas.xmlsoap.org/soap/envelope/")
  • Body: Contains the actual call/response data (e.g., <m:calculateCarPayment> with loanAmount, loanTerm, loanRate)

Infrastructure as a Service (IaaS)

Virtualized Computing Resources on Demand

All cloud computing services provide access to computing resources in a virtualized environment, 'the Cloud', across a public connection, usually the internet.
IaaS delivers virtualized hardware—virtual machines, storage, networking—over the internet or private networks.

IaaS enables users to access computing infrastructure as if they owned it, while the provider manages physical hardware in data centers.

Core Characteristics of IaaS

  • Self-service provisioning: Users independently request and deploy resources without IT intervention.
  • Virtualization: All infrastructure components (CPU, memory, storage, networking) are abstracted and pooled.
  • Dynamic scaling: Resources automatically scale up or down based on demand.
  • Metered usage: Customers pay only for consumed resources—CPU, memory, storage—measured per instance-hour.

Self-Service as a Business Enabler

The banking ATM service is a great example of the business value of self service. Without the availability of the self-service ATM, banks would be required to use costly resources to manage activities of all their customers—even for the most repetitive tasks.

Self-service in IaaS mirrors this: users make direct requests for routine infrastructure tasks, conforming to predefined business rules, eliminating manual overhead.

  • Users bypass IT bottlenecks for VM provisioning, storage allocation, and network configuration.
  • Reduces operational cost and accelerates time-to-service.
Self-service is imperative—not optional—in IaaS design and user experience.

Hybrid Cloud & Cloud Bursting

Hybrid cloud combines private/internal and external cloud resources to optimize flexibility and resource allocation.

  • Private clouds: Internal infrastructure behind firewall, used for sensitive data and day-to-day operations.
  • Public clouds: External providers offering scalable, on-demand infrastructure.
  • Cloud bursting: Offloading workloads to public cloud during demand spikes, releasing resources when demand drops.

Virtualization & VM Lifecycle Management

Virtualization enables infrastructure abstraction, allowing VMs to be managed as portable data files—facilitating migration, automation, and optimization.

  1. VM request → IT reviews resource pool → provisioning begins.
  1. VM provisioned and started → service delivered per SLA.
  1. VM released after lifecycle ends → resources freed for reuse.
  • Live migration: Moves powered-on VMs between hosts in milliseconds—no user impact.
  • Use cases: Proactive maintenance, load balancing, zero-downtime updates.
VM migration tools like VMware VMotion and Citrix XenMotion enable seamless mobility.

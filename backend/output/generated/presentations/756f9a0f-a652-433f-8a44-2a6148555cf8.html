Distributed Systems & Cloud Computing

From RPC to Web Services and IaaS

  • Distributed systems hide heterogeneity via transparency
  • Middleware enables cross-platform communication
  • Web services standardize interoperability over HTTP/XML
  • IaaS delivers infrastructure as a utility-based service

This presentation explores the evolution from local/remote procedure calls to modern cloud infrastructure, emphasizing how distributed computing principles underpin cloud service models.

Distributed Systems Fundamentals

A distributed system is a collection of independent computers that appears to users as a single coherent system. It enables resource sharing, collaboration, and fault tolerance.

Remote Procedure Call (RPC) Architecture

RPC enables a process to execute code on a remote machine as if it were local. It uses stubs and skeletons to marshal/unmarshal parameters and manage network communication.

  • Client stub serializes parameters and sends message over network
  • Server stub receives message and deserializes parameters
  • Server executes the requested function
  • Response follows reverse path: server stub → network → client stub
  • Client resumes execution with returned value

Web Services: Architecture & Standards

Web services solve interoperability, firewall traversal, and complexity issues of earlier distributed systems by using HTTP and XML-based protocols.

  • Self-contained, platform-independent, loosely coupled
  • Use WSDL to describe interfaces, UDDI for service discovery
  • SOAP for structured messaging, REST for lightweight access
  • Accessible over standard HTTP (port 80), enabling firewall-friendly communication

Web Services Lifecycle & Registry

Web services follow a publish-find-bind-invoke pattern. UDDI acts as a directory service, enabling dynamic discovery and integration across organizations.

  • Publish: Service provider registers in UDDI with WSDL
  • Find: Client queries UDDI to locate suitable services
  • Bind: Client retrieves WSDL and binds to service endpoint
  • Invoke: Client sends SOAP messages to execute operations

XML in Web Services

XML is foundational to web services: it enables platform-independent data exchange, supports validation via schemas, and allows structured messaging for interoperability.

  • Well-formed syntax ensures correct tag nesting and closure
  • Valid XML conforms to XSD or DTD schema definitions
  • Supports simple types (int, boolean, date) and complex user-defined types
  • Parsers (DOM, SAX) enable programmatic processing

Cloud Computing: Core Concepts

Cloud computing shifts computing from local devices to shared, virtualized resources accessible over the internet. It emphasizes elasticity, self-service, and utility-based pricing.

  • Virtualization decouples OS from hardware, enabling migration and consolidation
  • Essential characteristics: on-demand self-service, broad network access, resource pooling, rapid elasticity, measured service
  • Service models: IaaS, PaaS, SaaS
  • Deployment models: public, private, community, hybrid

Virtualization & Hypervisors

Virtualization abstracts hardware resources into virtual machines. Hypervisors manage this abstraction, with Type 1 (bare-metal) offering better performance than Type 2 (hosted).

Infrastructure as a Service (IaaS)

IaaS delivers virtualized computing resources over the internet. It abstracts physical infrastructure, enabling scalable, on-demand access to VMs, storage, and networking.

  • Key features: dynamic scaling, utility pricing, self-service provisioning, metered usage
  • Examples: Amazon EC2, Microsoft Azure, Google Compute Engine, Rackspace
  • Supports enterprise infrastructure, cloud hosting, and virtual data centers
  • Enables VM migration (live, cold, storage) for maintenance and load balancing

XML vs HTML: Purpose and Structure

XML vs HTML: Purpose and Structure

  • XML: Content-oriented, extensible tags, designed to transport/store data
  • HTML: Presentation-oriented, fixed tags, designed to display data

XML allows users to define custom tags and attributes to describe data semantics, while HTML uses a rigid set of predefined tags to control visual presentation. XML is not intended for direct human consumption but for machine processing and data exchange.

XML Document Structure and Validation

XML Document Structure and Validation

An XML instance document includes an XML declaration, optional namespace declarations, and a hierarchical structure of elements. Unlike HTML, XML supports schema-based validation (e.g., via XSD), enabling strict data integrity checks.

SOAP Message Structure

SOAP Message Structure

SOAP (Simple Object Access Protocol) messages use XML syntax to encode remote procedure calls and responses. A SOAP message consists of an Envelope, Header (optional), Body (required), and Fault (for errors).

Interoperability Across Heterogeneous Environments

Distributed systems must achieve openness by abstracting away differences in underlying hardware, platforms, and languages. Standards like XML, SOAP, CORBA, and DCOM enable this interoperability.

Full Virtualization Techniques

Full Virtualization Techniques

  • Uses binary translation for privileged/sensitive instructions
  • Direct execution for user-level instructions
  • No guest OS modification required
  • Complete hardware simulation

Full virtualization provides a total abstraction of the physical system, allowing unmodified guest operating systems to run unaware of the virtual environment. This approach incurs overhead due to instruction translation but ensures compatibility.

XML vs HTML, SOAP, and Interoperability Foundations

XML vs HTML: Core Differences

  • XML: content-oriented, extensible tags, designed to transport/store data, supports validation
  • HTML: presentation-oriented, fixed tag set, designed to display data, no validation

XML Document Structure

An XML instance includes an XML declaration (e.g., <?xml version="1.0" encoding="UTF-8"?>), optional namespace declarations (e.g., xmlns, xsi:schemaLocation), and hierarchical, user-defined elements. Note: The example in the source contains syntax errors (e.g., missing quotes, malformed tags), but illustrates typical structure.

SOAP Message Structure

  • Envelope: root element defining namespace (SOAP-ENV)
  • Body: contains the actual request/response (e.g., <calculateCarPayment>)
  • Encoding: uses SOAP-ENC and XSD types (e.g., xsi:type="xsd:string")

Distributed System Interoperability Requirements

  • Interoperability across: operating systems, hardware architectures, communication architectures, programming languages, software interfaces, security measures, and information representation

Achieving Openness

  • Independence from heterogeneity of underlying environment: hardware, platforms, and languages

Legacy Middleware Standards

  • CORBA (Common Object Request Broker Architecture)
  • DCOM (Distributed Component Object Model)

Distributed Systems: Core Principles

Transparency, Openness, and Heterogeneity Masking

  • A distributed system is one on which I cannot get any work done because some machine I have never heard of crashed.
  • Desired properties: Transparency, Openness, Scalability, Enhanced Availability, Concurrency, Independent failures, Heterogeneity.

Distributed systems must abstract away underlying complexity to enable seamless interaction across heterogeneous environments.

Heterogeneity Masking via Middleware

  • Middleware frameworks mask heterogeneity of networks and hardware.
  • Most also mask heterogeneity of operating systems or programming languages, or both.
  • CORBA and similar frameworks mask heterogeneity among vendor implementations of the same middleware standard.

Middleware ensures that distributed systems remain independent of underlying hardware, platforms, and programming languages.

Openness and Interoperability Requirements

  • Systems should conform to well-defined interfaces.
  • Systems should support portability of applications.
  • Systems should easily interoperate with services from other open systems, irrespective of the underlying environment.

XML vs. HTML: Structural and Functional Differences

FeatureXMLHTML
TagsExtensible—user-definedFixed—pre-determined and rigid
PurposeTransport and store dataDisplay data and describe appearance
OrientationContent-orientedPresentation-oriented
ValidationSupports DTD and XML SchemaNo built-in validation
Data TypesSupports custom and predefined typesNo data typing

XML is designed for data transport and storage with rich validation; HTML is designed for human-readable presentation.

XML Schema and Validation

  • XML Schema is itself an XML document.
  • Supports predefined simple types (integers, booleans, dateTime) and user-defined complex types.
  • Provides validation capabilities including restrictions on types.
  • Alternative to DTD with added functionality and richer data typing.

XML Schema enables robust, type-safe validation of XML instances, unlike DTD.

XML Processing Interfaces

  • SAX (Simple API for XML): Event-based parsing.
  • DOM (Document Object Model): Loads XML into memory as a tree structure.
  • Python implements these via the PyXML package.

Middleware Examples and Grid Computing

  • CORBA: Common Object Request Broker Architecture.
  • DCOM: Distributed Component Object Model.
  • Grid Computing: Uses loosely coupled, heterogeneous, geographically dispersed clusters.

Middleware frameworks like CORBA and DCOM abstract platform and language differences; Grid Computing extends this across distributed, heterogeneous infrastructures.

Virtualization Trade-offs

  • Enables abstraction of underlying hardware and OS.
  • Disadvantage: Requires modification of guest OS kernel to communicate with hypervisor.
